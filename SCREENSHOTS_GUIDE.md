# Чеклист скриншотов для кейсов

Рекомендуемый формат: **PNG**, ширина **1600px** (или 1920px), соотношение сторон **16:9**.

## Общие правила
- Скрывай чувствительные данные (токены, email, приватные id).
- Держи единый визуальный стиль: одинаковый масштаб браузера и чистый интерфейс.
- На один кейс достаточно **3-5 скриншотов**.

## UnTT (Telegram-бот)
- Основной сценарий чата: `/start` + ключевые команды.
- Пример проверки тайминга (ввод -> ответ бота).
- Пример уведомления от scheduler.
- Опционально: небольшой фрагмент админ/конфиг-экрана.

## FA (Demo / MVP)
- Главный экран / входная точка.
- Сценарий создания задачи или workflow.
- Пример данных в базе/админ-представлении.
- Любая реализованная автоматизация в действии.

## Price Tracker Portfolio
- Список отслеживаемых товаров.
- История изменения цены (до/после).
- Пример срабатывания уведомления.
- Экран настроек источников или параметров отслеживания.

## Restaurant Terrassa
- Главный экран (hero-блок).
- Блок/страница меню.
- Раздел контактов/бронирования.
- Скрин мобильной адаптации.

## VG2 (демо генерации видео)
- Экран/конфиг входных параметров.
- Шаг выполнения пайплайна (статус процесса).
- Итог: список артефактов или кадр результата.
- Опционально: фрагмент runtime-логов для доверия.
