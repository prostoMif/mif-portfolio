# DNS для mif-portfolio.com (Railway)

Записи добавляются **у регистратора домена** (где куплен `mif-portfolio.com`: Cloudflare, Namecheap, REG.RU и т.д.), не в репозитории.

## Записи от Railway

| Type  | Name (Host)      | Value |
| ----- | ---------------- | ----- |
| CNAME | `@` (или пусто / корень домена) | `26l3s4vv.up.railway.app` |
| TXT   | `_railway-verify` | см. ниже |

### TXT (верификация)

**Имя:** `_railway-verify`  
**Значение:** скопируй **целиком** из панели Railway (раздел Custom Domain → Verify). Обычно формат такой:

```text
railway-verify=e952055a4ad3ac73912c4eab511569da06bf8097b23e4a60c9d2407ec607309a
```

Если в Railway показано так (как у тебя в кабинете) — вставь **без изменений**:

```text
railway-verify=railway-verify=e952055a4ad3ac73912c4eab511569da06bf8097b23e4a60c9d2407ec607309a
```

Если верификация не проходит — открой Railway снова и скопируй TXT **заново** (токен мог обновиться).

## После сохранения записей

1. Подожди **5–60 минут** (иногда до 24–48 ч при кэше DNS).
2. В Railway нажми **Verify** / проверь статус домена.
3. Railway сам выдаст **SSL** (HTTPS), когда DNS подтянется.

## Корень домена `@` и CNAME

У части регистраторов для **корня** (`@`) нельзя поставить обычный CNAME по правилам DNS. Тогда:

- **Cloudflare:** CNAME на `@` с прокси (оранжевое облако) часто работает.
- Или используй **ALIAS / ANAME** на целевой хост Railway, если регистратор предлагает.
- Или перенеси NS на **Cloudflare** и настрой там по их инструкции.

Если что-то не верифицируется — смотри актуальные записи в **Railway → Settings → Networking → Custom Domain** и сравни с панелью DNS.
